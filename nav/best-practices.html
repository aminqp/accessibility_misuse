Accessibility Best Practices for the Nav Element

Best Practice: Keyboard and Screen Reader Accessibility

Key Accessibility Guidelines for Nav Elements

1. Clear Labeling

Each nav element should have a descriptive label:

<nav aria-label="Primary navigation">
    <!-- Navigation content -->
</nav>

Or use aria-labelledby to reference a visible heading:

<h2 id="main-nav-heading">Main Navigation</h2>
<nav aria-labelledby="main-nav-heading">
    <!-- Navigation content -->
</nav>

2. Proper Structure

Use unordered lists for navigation menus:

<nav aria-label="Main navigation">
    <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Contact</a></li>
    </ul>
</nav>

3. Keyboard Navigation

Ensure all navigation is fully keyboard accessible:

  • All links must be reachable with the Tab key
  • Dropdown menus should be operable with keyboard (Enter/Space to open, arrow keys to navigate)
  • Visible focus indicators should be present for all interactive elements

4. Skip Navigation Link

Add a skip link at the beginning of the page to allow keyboard users to bypass repetitive navigation:

<a href="#main-content" class="skip-link">Skip to main content</a>
<!-- Navigation here -->
<main id="main-content">
    <!-- Main content -->
</main>

With CSS to show the skip link only when it receives focus:

.skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    background: #000;
    color: white;
    padding: 8px;
    z-index: 100;
}
.skip-link:focus {
    top: 0;
}

5. Current Page Indication

Clearly indicate the current page in navigation:

<nav aria-label="Main navigation">
    <ul>
        <li><a href="/" aria-current="page">Home</a></li>
        <li><a href="/about">About</a></li>
        <li><a href="/contact">Contact</a></li>
    </ul>
</nav>

6. Mobile Navigation Considerations

For hamburger menus or collapsible navigation:

  • Use aria-expanded to indicate the open/closed state
  • Use aria-controls to associate the button with the navigation it controls
  • Provide a clear button label
<button aria-expanded="false" aria-controls="mobile-nav" aria-label="Menu">
    <span>☰</span>
</button>
<nav id="mobile-nav" aria-label="Main navigation" hidden>
    <!-- Navigation content -->
</nav>
