Issue: Hidden Main Element

Issue 4: Hidden main element

Website Title

Main Content

This main content is hidden and thus inaccessible to all users.

Visible Content

This is the content users actually see, but it's not in a main element.

Footer content

Problem: Hiding the <main> element with the hidden attribute while displaying the primary content elsewhere creates confusion for screen reader users who rely on the <main> element to find the primary content.

Correct Implementation

Website Title

Main Content

This main content is visible and accessible to all users.

Additional Content

This content is also within the main element, making it properly structured and accessible.

Footer content

Solution: Ensure that the <main> element contains all the primary content and is visible to users. If content is meant to be the primary content of the page, it should be inside the main element and not hidden.

Appropriate Use of Hidden Content

While you should not hide the main element itself, there are legitimate cases for hiding content within a page:

  • Single Page Applications: When switching between views, you might temporarily hide one main element and show another (but not at the same time)
  • Progressive Disclosure: Using details/summary or expandable sections to show additional content when needed
  • Screen Reader Only Text: Providing extra context for screen reader users without affecting visual presentation

Example: Legitimate Use of Hidden Content

<main id="main-content">
    <h1>Product Information</h1>

    <!-- Visually hidden instructions for screen reader users -->
    <p class="sr-only">Use the tabs below to navigate product information.</p>

    <div class="tabs">
        <button aria-selected="true" id="tab1">Description</button>
        <button aria-selected="false" id="tab2">Specifications</button>
    </div>

    <div id="panel1" aria-labelledby="tab1">
        <h2>Product Description</h2>
        <p>Product details here...</p>
    </div>

    <div id="panel2" aria-labelledby="tab2" hidden>
        <h2>Product Specifications</h2>
        <p>Technical specifications here...</p>
    </div>
</main>

In this example, the main element itself is visible, but some content within it is conditionally shown or hidden based on user interaction. This is an appropriate use of hidden content.
