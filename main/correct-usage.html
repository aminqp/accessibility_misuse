Why this is good:

  • There is exactly one <main> element on the page
  • The <main> element is not nested inside another landmark element
  • It contains the primary content of the document
  • It has an ID for "skip to content" links
  • It has tabindex="-1" to allow it to receive focus programmatically
  • Secondary content (navigation, complementary content) is outside the main element
  • All navigational elements have appropriate ARIA labels
  • The main content is preceded by a skip link for keyboard users
Skip Link Implementation Details

For a proper skip link implementation:

<header>
    <!-- Header content -->
    <a href="#main-content" class="skip-link">Skip to main content</a>
</header>

<main id="main-content" tabindex="-1">
    <!-- Main content -->
</main>

CSS for the skip link:

.skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    background: #000;
    color: white;
    padding: 8px;
    z-index: 100;
}

.skip-link:focus {
    top: 0;
}

This creates a skip link that is visually hidden until it receives focus from keyboard navigation, at which point it becomes visible for users who need it.
