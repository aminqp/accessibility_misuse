Landmark Element Hierarchy

The HTML5 specification defines a specific hierarchy for landmark elements to ensure consistent document structure and accessibility:

Proper Nesting of Landmarks

  • <body>
    • <header>
    • <nav>
    • <main>
      • <article>
      • <section>
    • <aside>
    • <footer>

Technical Reasons for Proper Structure

According to the HTML5 specification, the <main> element has content restrictions. It must not be a descendant of an <article>, <aside>, <footer>, <header>, or <nav> element. This is because:

  • The <main> element is intended to contain the primary content of the page
  • Other landmark elements are intended for specific types of content (navigation, complementary, etc.)
  • Nesting main within other landmarks creates conflicting semantics about what constitutes the "main" content
  • Screen reader users navigate by landmarks, and proper hierarchy is essential for effective navigation
