Issue: Footer Inside Generic Container

Issue 5: Footer inside a section without sectioning context

Website Title

This is some content in a generic div...

This footer relates to the div above, but the relationship is unclear.

Problem: When a footer is placed inside a generic container like a div, the relationship between the footer and its parent content is not semantically clear. Footers should be inside sectioning elements (article, section, aside, nav) or at the document level.

Correct Implementation

Website Title

Section Title

This is some content in a properly structured section...

This footer clearly relates to the section above.

Solution: Place footers inside proper sectioning elements (article, section, aside, nav) to create a clear semantic relationship between the footer and its related content. This helps assistive technologies understand the structure and relationship of the content.

/* Another good example: footer in an article */
<article>
    <h2>Article Title</h2>
    <p>Article content...</p>
    <footer>
        <p>Posted by: Author Name</p>
    </footer>
</article>
