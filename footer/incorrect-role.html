Problem: Applying the "main" role to a footer creates confusing and contradictory semantics. This tells assistive technologies that this content is both footer content and the main content of the page.

Correct Implementation

Solution: Allow the footer element to use its default implicit "contentinfo" role (when at the document level) without overriding it. The role="main" should only be used on the element containing the primary content of the page, typically the main element.

Note: A footer element that is a direct descendant of the body has an implicit "contentinfo" landmark role; a footer inside an article or section doesn't get any landmark role automatically.
