Problem: Multiple footers at the document level both with contentinfo role creates confusion for screen reader users. The contentinfo role (which is implicit for footer elements at the document level) should be used only once per page.

Solution: Use a single footer element at the document level, organizing different types of footer content within it. If you need visual separation, use divs or other elements inside the footer rather than creating multiple footer elements.

Alternatively, if you need multiple footer-like sections, only the main document-level footer should be a direct child of the body element:

/* Alternative approach with secondary footer-like content */
<div class="secondary-info">
    <p>Additional information that looks like a footer but isn't the main site footer</p>
</div>
<footer>
    <p>Main document footer with contentinfo role</p>
</footer>
