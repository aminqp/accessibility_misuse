Problem: Using header for metadata or footer information is semantically incorrect. This information should be in a footer element. This misuse confuses screen reader users who expect headers to contain introductory content, not concluding metadata.

Solution: Use the footer element for metadata, attribution information, and similar content that appears at the end of a section. Reserve the header element for introductory content that appears at the beginning of a section. This creates a logical document structure that matches user expectations and is clearer for assistive technologies.
