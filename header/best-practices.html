Accessibility Best Practices for the Header Element

Header Accessibility Guidelines

Key Best Practices for Accessible Headers

1. Include a Heading Element

Always include at least one heading element (h1-h6) within a header to provide context and structure:

<header>
    <h1>Website Title</h1>
    <!-- Other header content -->
</header>

2. Provide Skip Links

Include a skip link at the beginning of the page to allow keyboard users to bypass repetitive header content:

<a href="#main-content" class="skip-link">Skip to main content</a>
<header>
    <!-- Header content -->
</header>
<main id="main-content">
    <!-- Main content -->
</main>

With CSS to make the skip link visible when focused:

.skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    background: #000;
    color: white;
    padding: 8px;
    z-index: 100;
}
.skip-link:focus {
    top: 0;
}

3. Use Appropriate Landmark Roles

The main page header typically gets the "banner" role automatically (when it's a direct child of the body element). For other headers, no additional role is needed:

<!-- Main page header (automatically gets banner role) -->
<header>
    <h1>Site Title</h1>
</header>

<!-- Section header (no banner role needed) -->
<section>
    <header>
        <h2>Section Title</h2>
    </header>
</section>

4. Keep Header Content Focused

Include only introductory or navigational content in headers:

  • Page title or section title (heading)
  • Logo
  • Primary navigation
  • Search function
  • Brief introductory text

5. Make Navigation Accessible

Ensure navigation menus within headers are fully accessible:

<header>
    <h1>Site Title</h1>
    <nav aria-label="Main navigation">
        <ul>
            <li><a href="#">Home</a></li>
            <li><a href="#">About</a></li>
            <li><a href="#">Contact</a></li>
        </ul>
    </nav>
</header>

6. Don't Nest Header Elements

Avoid nesting header elements inside each other. Instead, use appropriate sectioning elements:

<!-- Incorrect: Nested headers -->
<header>
    <h1>Main Title</h1>
    <header>
        <h2>Subtitle</h2>
    </header>
</header>

<!-- Correct: Proper sectioning -->
<header>
    <h1>Main Title</h1>
    <div>
        <h2>Subtitle</h2>
    </div>
</header>

7. Ensure Adequate Contrast

Make sure text in the header has sufficient contrast with the background (minimum ratio of 4.5:1 for normal text, 3:1 for large text). This is particularly important for headers where branding often introduces color challenges.

8. Make Headers Responsive

Ensure header components adapt well to different screen sizes, maintaining functionality and accessibility:

@media (max-width: 768px) {
    header {
        flex-direction: column;
    }

    nav ul {
        display: flex;
        flex-direction: column;
    }

    /* Ensure text remains readable */
    header h1 {
        font-size: 1.5em;
    }
}

9. Use Proper Header Semantics

Use the header element for its intended purpose - to contain introductory content for its parent section. Don't use it for decorative elements or containers that don't function as true headers.

10. Test with Assistive Technologies

Verify your header implementation by testing with screen readers to ensure:

  • The header is properly announced as a banner (for main page headers)
  • Skip links function correctly
  • Navigation is accessible and can be traversed with keyboard
  • The heading hierarchy is logical and navigable
Understanding Header Element Semantics

The Semantic Meaning of the Header Element

The <header> element represents introductory content for its nearest ancestor sectioning content or sectioning root element. It typically contains:

  • One or more heading elements (h1-h6)
  • Logo or icon
  • Author information
  • Navigation
  • Search forms

Different Types of Headers

Page-level Header

The main header of a page, typically containing the site title, logo, and main navigation:

<header>
    <h1>Website Name</h1>
    <nav>...</nav>
</header>

Section-level Header

A header that introduces a specific section of content:

<section>
    <header>
        <h2>Section Title</h2>
        <p>Brief description of this section.</p>
    </header>
    <!-- Section content -->
</section>

Article-level Header

A header for an article, typically containing the article title and metadata:

<article>
    <header>
        <h2>Article Title</h2>
        <p>Posted on: <time datetime="2023-01-01">January 1, 2023</time></p>
    </header>
    <!-- Article content -->
</article>

Landmark Role Considerations

Headers have special significance in the accessibility tree:

  • A <header> element that is a direct descendant of <body> automatically gets the "banner" landmark role
  • A <header> inside a sectioning element (article, section, etc.) does not get assigned any landmark role automatically
  • There should typically be only one "banner" landmark per page

This distinction helps screen reader users understand which header represents the main site header versus section-specific headers.
