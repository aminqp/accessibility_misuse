Problem: Multiple headers with the same landmark role (banner) create confusion for screen reader users. There should typically be only one banner per page. This violates the ARIA authoring practices and creates an ambiguous page structure.

Solution: Use the banner role only for the primary site header, and leave other header elements with their default role. For section-specific headers, don't add landmark roles. This creates a clear landmark structure for assistive technology users.

Note: The banner role is automatically assigned to a header element when it's a direct child of the body element, so you typically don't need to add role="banner" explicitly unless you're using a different element structure.
