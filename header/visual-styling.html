Issue: Using Header for Visual Styling Only

Issue 3: Using header for visual styling only

This content just needs to stand out visually

Problem: Using the header element purely for visual styling rather than semantic structure misuses the element's purpose. This confuses assistive technologies and screen readers, which expect header elements to contain introductory content for a section.

Correct Implementation

This content just needs to stand out visually

Solution: Use regular div elements with appropriate CSS classes for content that only needs visual distinction. Reserve the header element for actual introductory content of sections. This maintains the correct semantic meaning of header elements while still allowing for visual styling flexibility.

.highlighted-content {
    background-color: #333;
    color: white;
    padding: 20px;
}
