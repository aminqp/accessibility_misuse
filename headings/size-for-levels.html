Problem: Heading levels should represent the document structure, not be chosen based on their visual size. When headings are selected based on their default styling rather than their semantic meaning, the document structure becomes confused and difficult for screen reader users to navigate.

Solution: Choose heading levels based on their semantic meaning and document structure, then use CSS to adjust their visual appearance if needed. This approach maintains the correct document outline while allowing for design flexibility:

/* CSS to adjust heading sizes while maintaining semantics */
h1 {
    font-size: 1.8em;
}
h2 {
    font-size: 1.5em;
}
h3 {
    font-size: 1.3em;
}
