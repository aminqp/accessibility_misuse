Issue: Empty Heading Tags

Issue 7: Empty heading tags

Content after an empty heading...

Problem: Empty headings provide no information to screen reader users but are still announced as headings. This creates confusion and a poor user experience as screen readers will announce "heading level 2" without any content, leaving users to wonder what the section is about.

Correct Implementation

Section Title

Content with a proper heading...

Solution: Ensure all heading elements contain descriptive text that clearly identifies the content of the section they introduce. If a visual separator is needed without semantic heading meaning, use alternative elements like:

  • A styled hr element for a divider
  • A div with a border for visual separation
  • Margin or padding to create space
