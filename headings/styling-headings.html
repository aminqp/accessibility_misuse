Issue: Using Headings for Styling

Issue 2: Using headings for styling

This is not a true heading, just using h4 for its appearance

Problem: Using heading tags just to make text look a certain way is problematic. Headings should represent the document structure, not be chosen for their visual styling. This practice confuses screen reader users by announcing something as a heading when it doesn't actually serve as a structural element.

Correct Implementation

This is styled text, not a heading

Solution: Use CSS to style text that needs visual emphasis but isn't a true heading. Reserve heading elements (h1-h6) only for actual section headings that represent the document structure. If you need text to look like a heading for visual design, use CSS classes to achieve the desired appearance.

.styled-text {
    font-size: 1.2em;
    font-weight: bold;
    margin-bottom: 10px;
}
