Accessibility Best Practices for Headings

Heading Accessibility Guidelines

Key Best Practices for Accessible Headings

1. Use a Single H1

Each page should have exactly one h1 element that clearly identifies the page's main topic or purpose.

<h1>Page Title</h1>

2. Maintain Proper Hierarchy

Use headings in sequential order without skipping levels (h1 → h2 → h3). This creates a logical document outline.

<h1>Page Title</h1>
<h2>Major Section</h2>
<h3>Subsection</h3>
<h3>Another Subsection</h3>
<h2>Another Major Section</h2>

3. Use Headings for Structure, Not Styling

Select heading levels based on their semantic meaning, not their default appearance. Use CSS for styling:

/* Adjusting visual appearance while preserving semantics */
h2 {
    font-size: 1.5em;
    color: #0066cc;
}
/* Instead of using h4 just for its smaller size */

4. Make Headings Descriptive

Write clear, descriptive headings that accurately summarize the content that follows:

  • Good: <h2>How to Apply for Financial Aid</h2>
  • Poor: <h2>Information</h2>

5. Keep Headings Concise

Headings should be brief while still being descriptive. Avoid very long headings that are difficult to scan.

6. Ensure Proper Contrast

Maintain sufficient color contrast between headings and their background (minimum ratio of 4.5:1 for normal text, 3:1 for large text).

7. Don't Hide Headings

If a heading is needed for screen reader users but should be hidden visually, use accessible hiding techniques:

.visually-hidden {
    position: absolute;
    width: 1px;
    height: 1px;
    margin: -1px;
    padding: 0;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    border: 0;
}

/* Instead of display: none; which hides from all users */

8. Avoid Empty Headings

Always include text content in heading elements. Empty headings confuse screen reader users.

9. Don't Use Headings for Buttons or Links

Don't wrap headings around interactive elements:

  • Incorrect: <h2><button>Click Me</button></h2>
  • Better: <h2>Section Title</h2><button>Click Me</button>

10. Test with Screen Readers

Verify your heading structure by testing with screen readers and using the document outline feature in browser developer tools.

How Screen Reader Users Navigate with Headings

Understanding Heading Navigation

Screen reader users often navigate web pages by jumping from heading to heading. This is one of the most common navigation techniques because it allows users to quickly understand the page structure and find specific content.

Common Screen Reader Commands for Headings

  • JAWS and NVDA: Press H to move to the next heading
  • VoiceOver: Use Command + Option + H to move to the next heading
  • Most screen readers: Use numbers 1-6 to jump to specific heading levels (e.g., press 2 to jump to the next h2)

Why Proper Heading Structure Matters

When headings are used correctly:

  • Users can quickly scan the page structure
  • Navigation between sections becomes efficient
  • The relationship between content is clear
  • Users can build a mental map of the page

Improper heading structure forces users to read through all content linearly, creating a frustrating and time-consuming experience.
