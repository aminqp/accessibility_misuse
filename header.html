Header Element Accessibility Misuse Examples

Issue 1: Header with no heading element

Welcome to our website

This is important information in the header

Problem: This header lacks a heading element (h1-h6), which makes it difficult for screen reader users to identify the purpose of this section.

Issue 2: Multiple header elements in one section

First Header

Some content here...

Another Header in Same Section

More content...

Problem: Multiple header elements in the same section create confusion about which one is the primary header, affecting the logical structure.

Issue 3: Using header for visual styling only

This content just needs to stand out visually

Problem: Using the header element purely for visual styling rather than semantic structure misuses the element's purpose.

Issue 4: Nested header elements

Main Header

Sub Header

Problem: Nesting header elements is inappropriate and confuses assistive technologies about the document structure.

Issue 5: Header with excessive, non-header content

Website Title

Welcome to our website where we discuss many interesting topics.

Founded in 2010, our organization has been dedicated to...

Our mission is to provide the best information about...

Contact us at example@example.com

  • Home
  • About
  • Services
  • Contact

Problem: Overloading the header with too much content that should be elsewhere in the document structure makes it harder for users to find the main page content.

Issue 6: Using header for footer content

Categories: Web Design, Accessibility

Problem: Using header for metadata or footer information is semantically incorrect. This information should be in a footer element.

Problem: Multiple headers with the same landmark role (banner) create confusion for screen reader users. There should typically be only one banner per page.

This demonstrates proper usage where:

  • The main page header contains the site title and primary navigation
  • Article sections have their own headers with appropriate headings
  • Each header has a clear purpose and contains a heading element
  • The document follows a logical, semantic structure

Accessibility Best Practices for the Header Element

  • Always include a heading element (h1-h6) within a header
  • Use header elements to introduce content sections, not for styling
  • Avoid nesting header elements
  • Don't use multiple headers for the same section
  • Keep header content relevant to its section
  • Use the correct semantic elements (header vs footer)
  • Ensure only one header has the "banner" role per page
